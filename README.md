## Solar Panel AI

Solar Panel AI is a deep learning project for **automatic detection and classification of solar panel defects** from electroluminescence (EL) and RGB images.  
The system targets faults such as **bird droppings, dust, electrical damage, physical damage, snow coverage**, and other anomalies on photovoltaic modules.

This repository contains:
- **Classification models** – YOLO-based classifiers and a ResNet‑50 classifier (PyTorch / torchvision)
- **Object detection models** – YOLO‑based detectors and Faster R‑CNN + EfficientNet‑AE
- **Dataset utilities** – conversion and organization tools for multiple public and custom datasets (e.g. VOC → YOLO, custom → YOLO)
- **Analysis tools** – training metrics analysis, confusion matrices, and Grad‑CAM visualizations

All large datasets and model weights are kept locally and excluded from Git via `.gitignore`, so the GitHub repository stays lightweight and code‑focused.

---

### Repository structure

- `data/scripts/` – main Python scripts  
  - `train_yolov12_classification.py` – YOLO multi‑class fault classification  
  - `train_yolov12_object_detection.py` – YOLO‑based object detection training  
  - `train_yolov8.py`, `train_classification.py`, `train_detection_frcnn_ae.py` – additional YOLO / Faster R‑CNN + AE training variants  
  - `resnet train code .py` – ResNet‑50 classifier training (PyTorch / torchvision)  
  - `convert_*_to_yolo*.py`, `merge_yolo_dataset.py`, `organize_dataset.py` – dataset conversion and organization utilities  
  - `gradcam_produce.py` – Grad‑CAM visualization for trained classifiers  
  - `analyze_results.py` – analysis of YOLO training metrics (`results.csv`)  
- `data/detection_yolo*/` – YOLO detection datasets, configuration files, and helper scripts  
- `data/classification/` – EL and RGB classification datasets (**ignored in Git**)  
- `data/models/` – model package and local weight files (model checkpoints are ignored in Git)  
- `runs/` – training outputs, logs, and visualizations generated by Ultralytics (**ignored in Git**)  
- `ilkel dataset do not use for traning/` and other raw dataset folders – legacy / raw data used during experimentation (**ignored in Git**)

### Installation

- **Python**: 3.10–3.11 recommended  
- **Dependencies**: listed in `requirements.txt`

Install the project dependencies:

```bash
pip install -r requirements.txt
```

This installs:
- PyTorch and torchvision  
- Ultralytics (YOLO)  
- NumPy, Pandas, scikit‑learn  
- Matplotlib, Seaborn, OpenCV  

For the latest YOLO features (e.g. YOLOv12 from the Ultralytics repository), you can run:

```bash
python data/scripts/install_yolov12_from_github.py
```

This script upgrades the Ultralytics package directly from GitHub when available.

### Data

Datasets are **not stored in this repository**. They are expected to be placed locally with the following structure:

- Classification datasets:
  - `data/classification/el/...`
  - `data/classification/rgb/...`
- YOLO detection datasets:
  - `data/detection_yolo/...`
  - `data/detection_yolo_rgb_multiclass/...`

Several helper scripts in `data/scripts/` prepare external datasets:

- `convert_voc_to_yolo.py` – converts VOC XML annotations to YOLO text format  
- `convert_faulty_to_yolo_detection.py` – converts the Faulty Solar Panel dataset into YOLO detection format  
- `convert_rgb_multiclass_to_yolo_detection.py`, `merge_yolo_dataset.py` – additional merging and conversion tools  

The `.gitignore` is configured so that large datasets and generated artifacts remain local and are not pushed to GitHub.

### Training

All commands below are run from the project root.

```bash
# Multi‑class classification with YOLO (Ultralytics)
python data/scripts/train_yolov12_classification.py

# Object detection with YOLO
python data/scripts/train_yolov12_object_detection.py

# YOLOv8‑based training
python data/scripts/train_yolov8.py

# ResNet‑50 classification (torchvision)
python "data/scripts/resnet train code .py"

# Grad‑CAM visualization for a trained classifier
python data/scripts/gradcam_produce.py

# Analysis of YOLO training results (mAP, loss curves, stability, etc.)
python analyze_results.py
```

Each training script contains a configuration section at the top (paths, hyper‑parameters, model names) that can be adjusted for a specific dataset and hardware setup.

### Typical workflow

1. **Prepare data**  
   Organize EL and RGB images into the specified folder structure and, if necessary, convert external datasets to YOLO format using the provided conversion scripts.
2. **Install environment**  
   Create a Python environment and install dependencies:  
   ```bash
   pip install -r requirements.txt
   ```
3. **Train models**  
   Run the desired training script:  
   - YOLO classification / detection (`train_yolov12_*.py`, `train_yolov8.py`)  
   - ResNet‑50 classification (`"data/scripts/resnet train code .py"`)
4. **Evaluate and interpret models**  
   - Inspect the `runs/` directory for Ultralytics logs, best checkpoints, and plots (stored locally).  
   - Use `analyze_results.py` for numerical analysis of training performance.  
   - Use `gradcam_produce.py` to visualize which regions of the solar panels drive the model decisions.
